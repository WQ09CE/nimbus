# 日志分析器 (Log Analyzer) 设计规格说明书

## 1. 项目概述
本项目旨在开发一个轻量级的 Python 脚本 `log_analyzer.py`，用于自动化处理和分析日志文件。该工具能够快速识别日志中的严重问题，并提供日志级别的分布概览。

## 2. 核心功能
设计应包含以下核心功能点：

1.  **文件读取**：支持从命令行参数接收日志文件路径，并安全读取文件内容。
2.  **日志级别统计**：
    - 识别常见的日志级别：`INFO`, `DEBUG`, `WARNING`, `ERROR`, `CRITICAL`。
    - 统计每个级别出现的总次数。
3.  **错误提取**：
    - 专门提取所有 `ERROR` 级别的日志行。
    - 保留错误的详细信息，以便后续排查。
4.  **简报输出**：
    - 在终端输出结构化的文本简报。
    - 包含：文件概况、各级别计数统计、错误详情列表。

## 3. 技术设计

### 3.1 模块结构
脚本将采用模块化设计，主要包含以下部分：
- `parse_arguments()`: 处理命令行输入。
- `analyze_log(file_path)`: 核心逻辑模块，负责逐行扫描文件并进行正则表达式匹配。
- `generate_report(stats, errors)`: 格式化结果并输出。

### 3.2 匹配策略
使用 Python 的 `re` 模块进行模式匹配。
- 匹配模式参考：`^(\d{4}-\d{2}-\d{2}\s\d{2}:\d{2}:\d{2})?\s*(\w+): (.*)$`
- 捕获组：时间戳（可选）、日志级别、消息内容。

### 3.3 数据结构
- `stats`: 使用 `collections.Counter` 存储各级别计数。
- `errors`: 使用 `list` 存储提取到的 ERROR 详情字符串。

## 4. 输出示例
```text
=== 日志分析简报 ===
分析文件: example.log
--------------------
统计摘要:
- INFO: 120
- WARNING: 5
- ERROR: 2
--------------------
错误详情:
1. [2023-10-27 10:00:01] ERROR: Connection failed
2. [2023-10-27 10:05:22] ERROR: Timeout accessing database
====================
```

## 5. 错误处理
- 文件不存在：提示明确的错误信息并退出。
- 编码问题：默认使用 `utf-8`，支持回退到通用解码。
- 空文件处理：报告“未发现有效日志条目”。
